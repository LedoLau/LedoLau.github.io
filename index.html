巡维智联平台用户介绍与使用说明

平台重点解决以下问题：

  • 网络设备多，资产信息分散，缺少统一台账。
  • 巡检依赖人工登录设备，效率低且结果难追溯。
  • 设备异常发现不及时，缺少持续监控与告警。
  • 终端接入位置不清楚，排查 IP/MAC 对应交换机端口耗时。
  • 多部门、多用户共用系统时，缺少清晰的数据范围和操作权限控制。

1. 平台定位

巡维智联平台面向网络运维场景，提供设备台账、自动巡检、运行监控、终端定位、权限管理和机房位置管理能力。

平台重点解决以下问题：

  • 网络设备多，资产信息分散，缺少统一台账。
  • 巡检依赖人工登录设备，效率低且结果难追溯。
  • 设备异常发现不及时，缺少持续监控与告警。
  • 终端接入位置不清楚，排查 IP/MAC 对应交换机端口耗时。
  • 多部门、多用户共用系统时，缺少清晰的数据范围和操作权限控制。

2. 核心功能概览

功能模块 主要作用 典型使用人员
资产台账 维护网络设备、终端资产基础信息 资产管理员、网络管理员
巡检管理 对设备执行单台/批量/计划巡检，生成巡检记录与报表 网络运维人员
监控管理 按监控项持续采集设备状态，触发告警和通知 网络运维人员、值班人员
终端关联 通过 ARP/MAC 数据定位终端接入交换机和端口 网络运维人员、桌面支持人员
机房管理 管理地区、机房、机柜和设备上架位置 机房管理员
权限管理 管理用户、用户组、组织和数据范围 系统管理员

3. 登录与首页

3.1 登录

访问系统地址后进入登录页。

默认管理员账号：

首次部署后建议立即完成：

  • 修改管理员密码。
  • 创建组织。
  • 创建用户组。
  • 创建日常运维账号。
  • 按职责分配权限。

3.2 首页

登录后进入仪表盘。首页用于查看整体运行情况，例如设备数量、巡检概况、异常信息、监控告警等。

如果某些菜单不可见，通常是当前账号没有对应权限。

4. 资产台账使用说明

资产台账是巡检、监控、终端定位和机房管理的基础数据来源。

入口：

4.1 设备台账

设备台账用于维护网络设备、服务器等需要管理的资产。

常用操作：

  • 新增设备。
  • 编辑设备基础信息。
  • 配置登录方式、账号、密码、SNMP 参数。
  • 配置设备角色，例如网关、接入设备、网关&接入设备。
  • 测试连接，确认端口连通和登录是否成功。
  • 查看设备详情。
  • 导入/导出设备台账。

建议填写的关键字段：

  • 设备名称
  • 管理 IP
  • 设备角色
  • 厂商/型号
  • 登录协议与账号密码
  • SNMP 配置
  • 所属组织

注意：

  • 设备管理页面不展示巡检状态，巡检结果在巡检相关页面查看。
  • 网关设备和接入设备需要正确设置角色，终端定位依赖该角色采集 ARP/MAC 数据。
  • 如果设备同时承担网关和接入交换机职责，建议角色选择“网关&接入设备”。

4.2 终端台账

终端台账用于维护电脑、打印机、摄像头、门禁等终端资产。

常用操作：

  • 新增终端资产。
  • 维护终端名称、IP、MAC、使用人、部门、位置。
  • 导入/导出终端台账。
  • 绑定终端定位结果。
  • 查看终端历史轨迹。

注意：

  • 终端不强制要求先创建物理位置，可以先录入台账，后续补充位置。
  • 终端定位效果依赖终端 IP/MAC 与采集到的 ARP/MAC 数据是否匹配。

5. 巡检管理使用说明

巡检管理用于自动登录设备执行巡检命令，生成可追溯的巡检记录和报表。

入口：

常用子功能：

  • 巡检资产
  • 计划任务
  • 任务中心
  • 巡检记录
  • 异常统计
  • 周期报表
  • 巡检设置

5.1 单台巡检

适用场景：

  • 临时检查某一台设备。
  • 验证设备账号密码和巡检命令。
  • 排查单台设备异常。

操作步骤：

  1. 进入 巡检管理 -> 巡检资产。
  2. 找到目标设备。
  3. 点击单台巡检操作。
  4. 等待巡检完成。
  5. 查看巡检结果、输出内容和摘要。

期望结果：

  • 巡检成功时生成巡检记录。
  • 登录失败、命令失败、超时等情况会显示失败原因。

5.2 批量巡检

适用场景：

  • 对一批设备做统一巡检。
  • 交接班前检查。
  • 变更前后检查。

操作步骤：

  1. 进入巡检资产或批量巡检入口。
  2. 按设备组、角色、组织或筛选条件选择设备。
  3. 发起批量巡检。
  4. 在 任务中心 查看执行进度。
  5. 在 巡检记录 查看结果。

注意：

  • 批量巡检受并发数影响。
  • 并发过高可能导致设备压力或网络拥塞。
  • 如果设备账号配置错误，会在巡检结果中显示登录失败。

5.3 计划巡检

适用场景：

  • 每天、每周、每月自动巡检。
  • 固定时间生成巡检记录。

操作步骤：

  1. 进入 巡检管理 -> 计划任务。
  2. 新建计划任务。
  3. 选择设备范围。
  4. 配置执行周期和时间。
  5. 保存并启用任务。

运维建议：

  • 日常巡检建议安排在业务低峰期。
  • 首次配置计划任务后，建议先手动执行一次验证设备范围和账号配置。

5.4 巡检记录与报表

巡检完成后可以查看：

  • 单台巡检记录
  • 批量巡检记录
  • 巡检周期报表
  • 异常统计

记录中重点关注：

  • 巡检结果
  • 开始/结束时间
  • 耗时
  • 失败原因
  • 巡检摘要
  • 原始输出

6. 监控管理使用说明

监控管理用于持续采集设备状态，并根据规则产生告警。

入口：

当前菜单顺序：

  1. 监控项
  2. 监控总览
  3. 监控看板
  4. 告警中心
  5. 策略与通知

6.1 监控项

监控项是监控系统的核心。每个监控项定义“监控哪些设备、采集什么指标、如何判断异常”。

常见监控项：

  • Ping 连通性
  • SNMP OID 指标
  • 设备在线状态
  • 自定义阈值指标

新增监控项步骤：

  1. 进入 监控管理 -> 监控项。
  2. 点击新增监控项。
  3. 填写监控项名称。
  4. 选择监控协议，例如 Ping 或 SNMP。
  5. 选择设备范围。
  6. 设置采集周期、阈值、连续触发次数。
  7. 保存并启用。

配置建议：

  • 关键设备可以配置较短采集周期。
  • 普通设备采集周期不宜过短，避免增加设备压力。
  • SNMP 监控项需要设备 SNMP 参数正确。

6.2 监控总览

监控总览用于查看整体监控状态。

重点查看：

  • 设备在线情况
  • 活跃告警数量
  • 严重告警数量
  • 最近采集时间
  • 监控项运行状态

适用场景：

  • 值班人员快速判断当前网络状态。
  • 管理人员查看整体健康度。

6.3 监控看板

监控看板用于自定义展示重点监控数据。

常用方式：

  1. 进入 监控管理 -> 监控看板。
  2. 新建看板。
  3. 添加组件。
  4. 绑定设备和监控项。
  5. 保存后用于日常展示。

适用场景：

  • 大屏展示。
  • 重点设备监控。
  • 业务系统网络状态展示。

6.4 告警中心

当监控项判断异常时，会生成告警。

告警中心用于查看：

  • 告警设备
  • 告警级别
  • 告警内容
  • 首次触发时间
  • 最近触发时间
  • 当前状态

处理建议：

  1. 先查看严重告警。
  2. 确认设备是否真实异常。
  3. 查看对应监控项和采样数据。
  4. 处理完成后记录处置结果。

6.5 策略与通知

策略与通知用于配置告警通知方式。

常见配置：

  • 通知渠道
  • 告警策略
  • 通知日志

使用步骤：

  1. 配置通知渠道，例如 Webhook、邮件等。
  2. 配置告警策略。
  3. 触发告警后查看通知日志。

注意：

  • 通知失败时应查看通知日志中的 HTTP 状态和错误信息。
  • 生产环境建议先用测试告警验证通知链路。

7. 终端定位使用说明

终端定位用于根据网关 ARP 表和接入交换机 MAC 表，自动定位终端接入到哪台交换机、哪个端口。

正常使用时，系统会按“采集与关联策略”自动采集 ARP、自动采集 MAC，并自动执行关联。用户通常不需要每天手工点击采集或关联按钮。

手工按钮主要用于：

  • 首次部署后验证设备账号、SNMP 和采集链路。
  • 网络变更后立即刷新定位结果。
  • 排查某台终端定位不准或定位不到的问题。
  • 自动任务异常后临时补采集、补关联。

入口：

核心数据来源：

  • 网关设备 ARP 表：提供终端 IP 与 MAC 的对应关系。
  • 接入交换机 MAC/FDB 表：提供终端 MAC 与接入口的对应关系。
  • 端口角色：用于判断端口是否为接入端口。
  • 端口信息点映射：用于把交换机端口映射到工位、墙口、配线架。

7.1 使用前准备

需要先确认：

  • 设备台账中已录入网关设备。
  • 设备台账中已录入接入交换机。
  • 设备角色配置正确。
  • SNMP 参数可用。
  • 接入交换机接口已发现。
  • 端口角色已维护。

设备角色建议：

  • 网关设备：需要采集 ARP。
  • 接入设备：需要采集 MAC/FDB。
  • 网关&接入设备：同时采集 ARP 和 MAC/FDB。

7.2 自动采集与自动关联

入口：

管理员可以配置：

  • ARP 采集周期。
  • MAC 采集周期。
  • 关联周期。
  • 采集并发数。
  • SNMP 超时时间。
  • 历史保留天数。
  • 关联端口角色。

默认情况下：

  • 系统按周期自动采集网关 ARP 表。
  • 系统按周期自动采集接入交换机 MAC/FDB 表。
  • 系统按周期使用最近一次 ARP/MAC 数据自动生成关联结果。

注意：

  • 自动关联不会在每次关联前强制重新采集 ARP/MAC。
  • 如果关联周期到了，但 ARP/MAC 周期还没到，系统会使用最近一次已有采集数据。
  • 为保证定位准确，建议关联周期不小于 ARP/MAC 采集周期。

7.3 手工采集与手工关联

正常日常使用不需要人工点击，以下按钮用于验证和排障：

  • 立即采集ARP：立即从网关设备采集 ARP 表。
  • 立即采集MAC：立即从接入交换机采集 MAC/FDB 表。
  • 立即关联一次：立即使用最近一次 ARP/MAC 数据重新生成关联结果。

典型使用场景：

  1. 新增网关或接入交换机后，手工采集一次确认是否能拿到数据。
  2. 用户刚换工位或刚迁移端口后，手工采集并关联一次，快速刷新结果。
  3. 终端定位不到时，分别查看 ARP 地址表、MAC 地址表和采集任务列表，判断是哪一步没有数据。

注意：

  • 立即关联只做关联，不会自动先采集 ARP/MAC。
  • 如果采集数据是 50 分钟前的，也可以关联成功，但结果可能不是实时状态。
  • 若希望结果更准确，建议先采集 ARP 和 MAC，再执行关联。

7.4 查看定位结果

在 关联结果总览 中可以查看：

  • 终端名称
  • 终端 IP
  • 终端 MAC
  • 网关设备
  • 接入交换机
  • 接入口
  • 关联状态
  • 关联时间

在 关联历史 中可以查看：

  • 历史接入变化
  • 接入口信息
  • 变化标记
  • 关联说明

7.5 端口信息点映射

入口：

用途：

  • 把交换机端口和实际工位、墙口、配线架关联起来。
  • 让终端定位结果不仅能看到交换机端口，还能看到实际位置。

可维护字段：

  • 信息点编号
  • 工位
  • 墙口
  • 配线架
  • 备注

支持：

  • 按接入交换机分页查看端口。
  • 批量编辑。
  • Excel 导入。
  • Excel 导出。

8. 机房管理使用说明

机房管理用于维护物理位置和设备上架关系，帮助用户知道设备实际安装在哪个地区、机房、机柜和 U 位。

入口：

主要功能：

  • 物理位置管理
  • 机柜管理
  • 机柜占用管理

8.1 物理位置管理

物理位置管理用于维护基础位置数据。

常见对象：

  • 地区
  • 机房
  • 机柜

推荐维护顺序：

  1. 先创建地区，例如总部、分支机构、园区。
  2. 再创建机房，并关联所属地区。
  3. 最后创建机柜，并设置机柜编号、容量和所属机房。

说明：

  • 地区和机房是位置主数据，供设备台账、机柜管理和机柜占用使用。
  • 设备和终端台账不强制要求先创建物理位置，可以后续补录。

8.2 机柜管理

机柜管理用于维护机柜本身的信息。

常见字段：

  • 地区
  • 机房
  • 机柜名称或编号
  • 总 U 位
  • 已用 U 位
  • 空闲 U 位
  • 备注

使用场景：

  • 查看某个机房有哪些机柜。
  • 查看每个机柜容量。
  • 查看机柜已用和空闲 U 位。
  • 为后续设备上架提供基础数据。

注意：

  • 机柜主要适合网络设备、服务器等需要上架的资产。
  • 普通终端更适合维护到地区、工位、墙口或信息点，不建议按机柜 U 位管理。

8.3 机柜占用管理

机柜占用管理用于维护设备实际占用的机柜 U 位。

常见字段：

  • 设备
  • 地区
  • 机房
  • 机柜
  • 起始 U 位
  • 高度 U 位
  • 机柜 U 位
  • 面向
  • 状态
  • 备注

其中：

  • 起始U位：设备从哪个 U 位开始安装。
  • 高度U位：设备占用几个 U。
  • 机柜U位：系统自动展示占用范围，例如 5-6U。

使用步骤：

  1. 进入 机房管理 -> 机柜占用管理。
  2. 新增占用记录。
  3. 选择设备。
  4. 选择地区、机房和机柜。
  5. 填写起始 U 位和高度 U 位。
  6. 保存后查看机柜占用情况。

系统会校验：

  • U 位不能超过机柜容量。
  • 同一机柜、同一面向下有效占用不能重叠。
  • 编辑占用时会排除当前记录自身，避免误判冲突。

8.4 设备详情中的机柜位置

设备上架后，可以在设备详情中查看机柜位置。

可查看内容：

  • 地区
  • 机房
  • 机柜
  • 机柜 U 位
  • 上架状态
  • 面向
  • 备注

如果设备尚未上架，设备详情会显示未上架。

8.5 日常维护建议

新增设备时：

  • 可以先录入设备台账。
  • 再到机柜占用管理完成上架。

设备迁移时：

  • 先确认目标机柜空闲 U 位。
  • 再编辑机柜占用记录。
  • 保存后检查设备详情中的机柜位置是否同步。

设备下架时：

  • 建议通过机柜占用管理释放占用。
  • 如需保留历史，可在备注中记录下架原因和时间。

9. 权限管理使用说明

权限管理用于控制用户能看什么数据、能执行什么操作。

入口：

9.1 权限模型

平台权限由三部分组成：

  • 用户
  • 用户组
  • 组织与数据范围

简单理解：

  • 用户决定“谁登录系统”。
  • 用户组决定“能做什么操作”。
  • 组织和数据范围决定“能看哪些数据”。

9.2 管理员账号

admin 是系统内置管理员，权限最高。

管理员通常用于：

  • 初始化系统。
  • 创建组织。
  • 创建用户组。
  • 创建用户。
  • 分配权限。
  • 维护授权和系统更新。

建议：

  • 不要把 admin 作为日常运维账号长期共用。
  • 应为不同人员创建独立账号，便于追踪操作。

9.3 用户组

用户组用于批量管理权限。

建议按岗位创建用户组，例如：

  • 资产管理员
  • 巡检执行人员
  • 巡检管理员
  • 监控管理员
  • 终端定位管理员
  • 机房管理员
  • 系统管理员

一个用户可以加入多个用户组，最终权限为多个用户组权限的并集。

9.4 常见权限说明

权限方向 说明
资产权限 控制设备台账、终端台账的查看、编辑、导入导出
巡检权限 控制单台巡检、批量巡检、计划任务、巡检设置
监控权限 控制监控项、看板、告警、策略通知
终端关联权限 控制 ARP/MAC 采集、关联、端口映射、策略设置
机房权限 控制地区、机房、机柜和上架管理
系统权限 控制用户、用户组、组织、授权、系统更新等

9.5 组织与数据范围

组织用于隔离不同部门或区域的数据。

常见数据范围：

  • 仅本人
  • 本组织
  • 本组织及下级
  • 全部

示例：

  • 分支机构运维人员只看本机构设备。
  • 总部管理员可查看本组织及下级组织设备。
  • 系统管理员可维护全局配置。

9.6 推荐配置流程

首次部署建议按以下顺序配置：

  1. 使用 admin 登录。
  2. 创建组织结构。
  3. 创建用户组。
  4. 给用户组分配权限。
  5. 创建用户。
  6. 给用户绑定组织和用户组。
  7. 用新用户登录验证菜单和数据范围。
  8. 再导入设备和终端台账。

10. 日常运维建议

10.1 每日检查

建议每日查看：

  • 监控总览
  • 告警中心
  • 任务中心
  • 最近巡检记录
  • 终端关联采集任务结果

10.2 每周检查

建议每周检查：

  • 巡检周期报表
  • 异常设备趋势
  • 监控项运行状态
  • 通知日志是否有发送失败
  • 设备台账是否需要更新

10.3 变更后检查

网络变更后建议：

  • 更新设备台账。
  • 执行单台或批量巡检。
  • 重新发现接口。
  • 采集 ARP/MAC。
  • 执行终端关联。
  • 更新机柜占用记录。
  • 检查告警是否恢复正常。

11. 常见问题

11.1 为什么看不到某个菜单？

通常是当前账号没有对应权限。请联系管理员检查用户组权限。

11.2 为什么巡检失败？

常见原因：

  • 设备 IP 不通。
  • 登录协议未启用。
  • 用户名或密码错误。
  • 提权命令或提权密码错误。
  • 设备提示符未适配。
  • 巡检命令不适用于当前厂商或型号。

11.3 为什么监控没有数据？

常见原因：

  • 监控项未启用。
  • 设备未绑定到监控项。
  • SNMP 参数错误。
  • 采集周期未到。
  • 设备不支持对应 OID。

11.4 为什么终端定位不到端口？

常见原因：

  • 网关没有采集到 ARP。
  • 接入交换机没有采集到 MAC/FDB。
  • 设备角色配置错误。
  • 端口角色不在关联策略内。
  • 终端不在线或 MAC 已老化。
  • 接入交换机不支持当前 MAC 表 OID。

11.5 为什么机柜 U 位保存失败？

常见原因：

  • 起始 U 位或高度 U 位超过机柜容量。
  • 同一机柜、同一面向已有设备占用该 U 位。
  • 未选择机柜。
  • 选择的设备已经存在有效上架记录。

11.6 为什么系统更新检查失败？

常见原因：

  • 更新源地址不可达。
  • latest.json 路径不正确。
  • 平台或架构目录不存在。
  • latest.json 字段缺失。
  • 私有更新源需要访问令牌。
